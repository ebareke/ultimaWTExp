ultimaWTExp

FASTQ → differential expression, splicing, fusions, enrichment & reports

A production-grade, fully reproducible Nextflow DSL2 RNA-seq analysis platform.

GitHub Install Usage Architecture FAQ

Run it

nextflow run ebareke/ultimaWTExp \
    --input samplesheet.csv \
    --design sample_design.tsv \
    --fasta genome.fa --gtf genes.gtf \
    --organism human \
    -profile docker

On a cluster, swap in -profile slurm,apptainer. No tools to install — one container image carries the entire toolchain.

What you get

QC, end to end

FastQC, fastp, Picard, Qualimap, RSeQC, preseq, dupRadar → one MultiQC report.

Flexible alignment

STAR (2-pass, chimeric) and HISAT2; featureCounts/HTSeq/STAR + Salmon quantification.

Differential expression

Metadata-driven DESeq2 (+ edgeR), multifactor/paired/batch models, full diagnostic plots.

Fusions & splicing

STAR-Fusion and rMATS (SE/RI/A5SS/A3SS/MXE).

Functional biology

clusterProfiler ORA (GO/KEGG/Reactome) and optional GSEA.

Four reports

Executive, Technical, QC and Differential HTML for every audience.

Reproducible by construction

Immutable Docker/Apptainer image, version-pinned toolchain, per-run software-version capture, SHA-256 checksums over deliverables, and a tool-free -stub-run that validates the whole topology in CI. Same inputs + same container = same outputs.

# Validate the entire pipeline with no tools or genomes:
python3 examples/scripts/make_synthetic.py
NXF_SYNTAX_PARSER=v1 nextflow run main.nf -profile test -stub-run
